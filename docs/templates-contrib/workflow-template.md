# General Contribution Workflow Template

Purpose:

- Give contributors a clear process for changes.

Steps:

1. Identify target area: contracts, services, modules, runtime, or external tie-in.
2. Add or update files under the correct folder.
3. Keep implementation in the owner area.
4. Use public boundaries between areas.
5. Run validation checks if available.
6. Update docs if new boundaries are introduced.

Checklist:

- [ ] Area identified correctly
- [ ] Files placed in the right folder
- [ ] Boundaries respected
- [ ] Validation run
- [ ] Docs updated when needed
